# 🏗 Архитектура GG.Shell

## Архитектурные паттерны

---

## 1. Redux (Flux-архитектура)

Централизованное управление состоянием приложения через Redux Toolkit.

### Слайсы:

* `computersSlice` — компьютеры, сессии, зоны
* `usersSlice` — пользователи, пополнения баланса
* `salesSlice` — товары, корзина, продажи
* `journalSlice` — история операций
* `dashboardSlice` — статистика для дашборда

### Асинхронные действия:

Используется `createAsyncThunk` для всех API-запросов.

### Преимущества:

* Единый источник правды
* Предсказуемые изменения состояния
* Удобная отладка через Redux DevTools

---

## 2. Presentational & Container Components

Разделение компонентов по ответственности:

| Тип            | Расположение  | Назначение                                        |
| -------------- | ------------- | ------------------------------------------------- |
| Container      | `pages/`      | Логика, работа с Redux, запросы к API             |
| Presentational | `components/` | Отображение, принимают props, не зависят от Redux |

### Пример:

* `ComputersPage` (container) — получает данные из Redux, обрабатывает события
* `StartSessionModal` (presentational) — отображает форму, вызывает колбэки

---

## 3. Module Pattern

Изоляция логики в отдельные модули:

| Модуль               | Назначение                                          |
| -------------------- | --------------------------------------------------- |
| `api/client.js`      | Конфигурация Axios, перехватчики, работа с Supabase |
| `utils/dateUtils.js` | Работа с датами, часовыми поясами, расчёт стоимости |

### Преимущества:

* Переиспользование кода
* Лёгкая замена реализации
* Упрощение тестирования

---

## 4. Observer Pattern

Реализован через хуки React-Redux:

* `useSelector` — подписка на изменения состояния
* `useDispatch` — отправка действий

Компоненты автоматически перерисовываются при изменении данных в store.

---

## 5. Compound Components

Использование вложенных компонентов Ant Design:

```jsx
<Layout>
  <Sider>...</Sider>
  <Layout>
    <Header>...</Header>
    <Content>...</Content>
  </Layout>
</Layout>

<Modal>
  <Form>...</Form>
</Modal>
```

### Преимущества:

* Декларативное описание структуры
* Гибкая композиция

---

## 6. Higher-Order Components (HOC)

| HOC            | Назначение                       |
| -------------- | -------------------------------- |
| ConfigProvider | Тема и локализация Ant Design    |
| ErrorBoundary  | Отлов ошибок в графиках Recharts |

---

## 7. Custom Hooks

| Хук       | Назначение                                     |
| --------- | ---------------------------------------------- |
| useMemo   | Мемоизация вычислений (фильтрация, статистика) |
| useEffect | Побочные эффекты (загрузка данных, таймеры)    |
| useState  | Локальное состояние компонентов                |

---

## 8. Adapter Pattern

`client.js` адаптирует Axios под специфику Supabase API:

* Добавление заголовков `apikey`, `Authorization`
* Формат URL: `https://project.supabase.co/rest/v1/table`
* Обработка ошибок Supabase

## 📁 Структура проекта

```text
src/
├── api/            # HTTP-клиент (Module)
├── components/     # Презентационные компоненты
├── pages/          # Контейнеры (Container)
├── store/          # Redux (Flux)
│   └── slices/     # Слайсы состояния
├── utils/          # Утилиты (Module)
├── App.jsx         # Корневой компонент
└── main.jsx        # Точка входа
```
